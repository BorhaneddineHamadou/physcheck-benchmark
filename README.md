# physcheck-benchmark

Independent evaluation corpus for [physcheck](https://github.com/BorhaneddineHamadou/oddpilot),
the layered physical-plausibility linter for ASAM OpenSCENARIO 1.x.

## Ground-truth model

The benchmark measures both error directions of the linter, using scenarios whose
content comes from **real recorded driving** (drone recordings, field tests,
measurement campaigns — provenance documented per corpus):

- **Specificity / true negatives.** Real-world-derived scenarios describe things that
  physically happened, so they are physically plausible by construction. Every
  error-clean file is a true negative; every error finding on them is a **candidate
  false positive** and is hand-triaged (a finding can still be correct when the
  *conversion* to OpenSCENARIO introduced an impossibility — the triage notes say
  which).
- **Sensitivity / false negatives.** Real data contains no violations, so the positive
  class is created by **mutation**: `mutate.py` seeds exactly one certainly-impossible
  physical violation per mutant (sun above the zenith, snowfall at +30 °C, 0-dimension
  bounding boxes, 150 m/s commands, spawns 2 km off the map, ...). Each mutation
  declares the rule family that must fire; a mutant where it does not is a **false
  negative**. Mutations only edit elements the real file already has, so mutants stay
  maximally realistic; the applicability matrix is in each `mutants/<name>/manifest.json`.

## Layout

```
corpora/<name>/        real .xosc (+ .xodr maps) + PROVENANCE.md + CORPUS.json
mutants/<name>/        generated by mutate.py (mirrors maps/catalogs; manifest.json)
run_benchmark.py       lints both sides, computes the confusion numbers
results/               results_<version>.json + table.md per physcheck release
```

## Protocol

```bash
pip install physcheck              # or: pip install -e <oddpilot>/physcheck
python3 mutate.py corpora/<name> mutants/<name>     # regenerate mutants
python3 run_benchmark.py                            # all corpora, all layers L0-L2
```

`run_benchmark.py` reports, per corpus: scenario files, true negatives, specificity,
error/warning findings by rule (candidate FPs listed in the results JSON for triage),
seeded mutants, detected mutants (a finding of the *expected* family, any severity;
error-severity detection is reported separately in the JSON), sensitivity, and the
full false-negative list. `results/table.md` is the summary embedded in physcheck's
README — re-run on every physcheck release and whenever a corpus is added.

Scoring is **strict by family**: a mutant only counts as detected when a rule of the
seeded violation's family fires, so accidentally noisy rules cannot inflate
sensitivity.

**Conversion-artifact triage.** Machine-converted corpora can carry impossibilities
introduced by the conversion itself (template metadata, under-covering converted
maps). Findings verified as such — evidence required in `results/TRIAGE_<ver>.md` —
are listed per corpus in `CORPUS.json` `artifact_rules`; the runner excludes them
from the tool-false-positive count but reports them fully. They are correct linter
output about the files; they are just not tool mistakes.

## Corpora

See `corpora/<name>/PROVENANCE.md` for the real-data pedigree, license, and any
triage notes of each corpus. `CORPUS.json` per corpus sets lint layers and an optional
shared `--map`.
